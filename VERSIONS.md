## v4.3

**ปุ่มสลับปีย้ายเข้าแถวหัว** จากที่เคยเป็นแถบแยกอยู่บนสุด ตอนนี้อยู่ในแถวเดียวกับโลโก้ ชิดขวาสุด (`h1` ใช้ `flex:1` ดันปุ่มไปชิดขวาโดยอัตโนมัติ) และเลิกเปิดแท็บใหม่ — ทั้งสองไฟล์นำทางในแท็บเดียวกันแล้ว จึงตัดไอคอน ↗ และ attribute `target`/`rel` ที่ไม่มีความหมายอีกต่อไปออก

**สีปุ่มที่เลือกเปลี่ยนจากทองเป็นขาว** เพื่อไม่ให้ปนกับปุ่มเมนู (`.snav-btn.active`) ที่ใช้สีทองอยู่แล้วในแถบถัดลงมา ปุ่มที่ยังไม่เลือกใช้กรอบจาง โปร่งแสงบนพื้นกรมท่า

**ป้ายเวอร์ชันรวมเข้าหัวเรื่องแล้ว** `.ver-chip` "v4.2 · เทียบ 67–69" ที่เคยแทรกหลังโลโก้ถูกถอดออก (ทั้ง element และ CSS) เลขเวอร์ชันย้ายไปอยู่ในข้อความหัวเรื่องแทน: **"ระบบสืบค้นข้อมูลงบประมาณ v4.3 · พ.ร.บ.งบประมาณรายจ่ายประจำปีงบประมาณ พ.ศ. 2567 - 2569"**

**ไฮไลท์คำค้นหาในชื่อกิจกรรม** หน้าค้นหาผลผลิต/โครงการ — ระหว่างตรวจพบว่ามีฟังก์ชัน `renderOutDeptDetail` ที่มีโค้ดไฮไลท์กิจกรรมอยู่แล้ว แต่**ไม่ถูกเรียกใช้จากที่ไหนเลย** (เป็นโค้ดค้างจากตอนปรับโครงสร้างใน v3.8 ที่เปลี่ยนไปใช้กล่องไล่ลึกร่วม `drillOut` แต่ลืมลบของเดิม) แก้โดยเพิ่มการรับคำค้นหาผ่าน `opts.term` ใน `drillOut` เฉพาะจุดที่เรียกจากหน้าค้นหาผลผลิต/โครงการ หน้าอื่น (จังหวัด/แผนงาน) ที่ใช้กล่องเดียวกันไม่ได้รับผลกระทบเพราะไม่ส่งค่านี้ พร้อมลบฟังก์ชันที่ไม่ถูกใช้งานทิ้ง (51 บรรทัด)

**ไฟล์ 2570.html ได้รับการปรับผัง (เฉพาะปุ่มสลับปี)** ให้ตรงกับ index.html ทุกประการ — ตำแหน่ง สี และการนำทางในแท็บเดียวกัน ส่วนหัวเรื่องและป้าย "ร่าง" ของไฟล์นั้นไม่แตะ เพราะใช้คำที่เหมาะสมกับบริบทของตัวเองอยู่แล้ว

## v4.0

### แก้ระยะขอบตอนพิมพ์

รายงานเดิมสั่ง `body{padding:0}` ใน `@media print` ทำให้ตัวหนังสือชิดขอบกระดาษ แก้เป็น

- `@page{margin:16mm 14mm}` กำหนดระยะขอบมาตรฐาน
- คงระยะห่างที่ตัวเอกสารอีก `6mm 4mm` เผื่อกรณีผู้ใช้เลือก "ไม่มีระยะขอบ" ในกล่องพิมพ์ ซึ่งจะยกเลิก `@page`
- หัวตารางซ้ำทุกหน้าเมื่อตารางยาวข้ามหน้า (`thead{display:table-header-group}`)
- ไม่ให้หัวข้อหลุดไปอยู่คนละหน้ากับเนื้อหา

### รายงานผลสัมฤทธิ์และตัวชี้วัด

ปุ่มใหม่ในแท็บผลสัมฤทธิ์ & KPI มีเฉพาะ PDF ไม่มี CSV

- ระบุชื่อกระทรวงและหน่วยงาน
- สรุปจำนวนตัวชี้วัด จำนวนที่มีเป้าหมายปี 2567 และจำนวนที่ไม่มีในปี 2569
- ตารางเป้าหมาย 3 ปี พร้อมกลุ่มตัวชี้วัดที่ยกเลิกแล้ว
- หน่วยงานที่ไม่มีตัวชี้วัดแสดงข้อความแทนตารางว่าง

ระหว่างทำได้แยก stylesheet ของรายงานออกเป็นค่าคงที่ `REPORT_CSS` ให้รายงานทั้งสองแบบใช้ร่วมกัน การแก้ระยะขอบครั้งเดียวจึงมีผลกับทุกรายงาน

## v3.2 — กิจกรรม×จังหวัด ปี 2569 และการแก้ไขตามที่แจ้ง

**เพิ่มข้อมูลกิจกรรม × จังหวัด ปี 2569** จากไฟล์ `budget_activity_ทุกจังหวัดและส่วนกลาง_2569.xlsx` (สกัดจาก PDF รายจังหวัด 78 ไฟล์) — `DB.act.ap` ขยายจาก 2 ปีเป็น 3 ปี ครอบคลุม 99.96% ของยอดประเทศปี 2569

**แก้ไข 3 จุดตามที่แจ้ง:**
1. **คอลัมน์งบรายจังหวัดตรงกันแล้ว** — เดิมแถวจังหวัดมี 2 คอลัมน์ (2567/2568) ทำให้ตัวเลขไม่ตรงกับแถวกิจกรรมด้านบนที่มี 3 คอลัมน์ ตอนนี้แสดงครบ 3 ปีตรงกัน
2. **ย้ายจำนวนกิจกรรม/จังหวัดไปใต้งบแต่ละปี** — จากเดิมอยู่หน้าชื่อผลผลิต ย้ายไปวางใต้ตัวเลขงบของแต่ละปี **แยกรายปี** (⚙️ กิจกรรม · 🗺️ จังหวัด) โดยปี 2569 ใช้ค่าที่เชื่อถือได้จากข้อมูลหลัก ส่วน 2567/2568 คำนวณจากข้อมูลกิจกรรม
3. **กล่อง "ไม่มีในปี 2569" แสดงชื่อแล้ว** — เดิมแสดงแต่ยอดเงิน เพราะเรียก `P()` ที่อ่าน pool หลักอย่างเดียว แก้เป็น `PY()` ที่อ่าน extension pool ด้วย (บั๊กเดียวกับที่เคยเจอตอนตัวชี้วัด) แก้ทั้งหน้าหน่วยงานและหน้าจังหวัด

**หมายเหตุ** ยอดรายจังหวัดปี 2569 ที่มีคือ 99.96% ส่วนที่เหลือคือรายการกองทุนที่เอกสารรายจังหวัดไม่ได้แจกแจง (ระบุไว้ในหมายเหตุไฟล์ต้นทาง) · ต้องเพิ่ม `ส่วนกลาง` ใน extension pool ของ payload เพราะไม่มีใน pool หลัก

## แนวทางการตั้งเลขเวอร์ชัน

```
v<major>.<minor>
  major — เปลี่ยนขอบเขตข้อมูล เช่น เพิ่มปีงบใหม่
  minor — เพิ่มหรือแก้ฟีเจอร์ภายในขอบเขตเดิม
```

ตัวอย่างที่จะเกิดขึ้นต่อไป

| เวอร์ชัน | เนื้อหา |
|---|---|
| ~~v2.1~~ | ~~เฟส 2~~ — เสร็จแล้ว |
| v2.2 | เฟส 3 — PDF รายงานเปรียบเทียบ, drill-down กระทรวง→หน่วยงาน |
| v3.0 | เพิ่มปีงบ 2570 |

## การนำขึ้น Google Sites

แนะนำให้ทำสองหน้าแยกกันช่วงแรก แล้วค่อยเปลี่ยนเมื่อ v2 นิ่งแล้ว

```
หน้าเดิม   → v1.0   (ผู้ใช้ทั่วไป)
หน้าทดสอบ → v2.0   (ทดลองใช้ภายใน)
```

เมื่อมั่นใจแล้วค่อยเปลี่ยน iframe ของหน้าเดิมไปชี้ v2.0 โดยเก็บ v1.0 ไว้เป็นทางถอย
